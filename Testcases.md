# Authentication/Sign in/Sign up Test Cases

## **Sign up**
- Verify that user is able to register to the website
- Verify that error is throwing when Password and Confirm password does not match while registering
- Verify that user is able to log in when the user login with already existing email
- Verify that user is not able to register when one of the fields missed
- Verify that email validation has there for a email field
- Verify that user sees the dashboard/similar page when registration is completed

## **Sign in**
- Verify that user is able to login via login button with correct email and password
- Verify that user is not able to login with correct email and wrong password
- Verify that user is not able to login with wrong email and correct password
- Verify that user is not able to login with wrong email and wrong password
- Verify that the user is seeing the dashboard/similar page when login is successful

## **Log out**
- Verify that the user is able to logout successfully while clicking on log out button

## **Forgot password functionality**
- Verify that user receives an email when he entered the email on the forgot password model
- Verify that error was shown when wrong email is entered
- Verify that user is able to create a new password
- Verify that user is able to login with email and newly created password
